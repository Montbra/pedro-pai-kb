# Problema de GPU Híbrida - NVIDIA RTX 4070 + AMD Radeon 780M

## Data
2026-03-17

## Hardware
- GPU Dedicada: NVIDIA GeForce RTX 4070 Max-Q (PCI 01:00.0)
- GPU Integrada: AMD Radeon 780M / HawkPoint1 (PCI 06:00.0)
- Monitor externo conectado na saída da RTX 4070

## Problema Original

### Sintomas
1. **X11 travava** após login no SDDM (necessário power reset)
2. **Wayland funcionava** mas com 176+ erros de framebuffer por sessão:
   ```
   GL_INVALID_ENUM error generated. Invalid <face>.
   Invalid framebuffer status: "GL_FRAMEBUFFER_INCOMPLETE_MISSING_ATTACHMENT"
   Failed to create framebuffer: Invalid argument
   ```
3. Travamentos aleatórios do sistema

### Causa Raiz
- EnvyControl configurado em **modo nvidia**
- Arquivo `/etc/X11/xorg.conf` forçando NVIDIA como primária e AMD inativa
- Conflito entre drivers: ambos carregados mas mal configurados
- Wayland tentando usar AMD enquanto configuração forçava NVIDIA

## Modos do EnvyControl

### Modo Nvidia (anterior)
- NVIDIA sempre ativa como GPU primária
- AMD desativada
- Maior consumo de energia
- **Problema:** Conflitos no Wayland, X11 travando

### Modo Híbrido (aplicado)
- AMD como GPU primária para display interno
- NVIDIA gerencia monitor externo
- Alterna entre GPUs conforme necessidade
- Consumo balanceado
- **Vantagem:** Resolve conflitos, suporta monitor externo na NVIDIA

### Modo Integrado (não aplicável)
- Apenas AMD ativa
- NVIDIA completamente desligada
- **Limitação:** Não funciona com monitor externo na NVIDIA
